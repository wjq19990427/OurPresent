# OurPresent 用户使用文档

本文档面向最终使用者，介绍系统的安装、核心操作流程和常见问题。

## 1. 使用前准备

### 1.1 安装依赖

```bash
uv sync
```

### 1.2 启动应用

```bash
uv run streamlit run main.py
```

启动后访问 `http://localhost:8511`。

如果你希望让同一局域网中的另一台设备访问，也可以使用本机 IP 加端口访问。

## 2. 第一次使用

1. 双方分别注册自己的账号
2. 记录各自注册后显示的用户 ID
3. 一方在账户页输入对方用户 ID 发起绑定
4. 另一方登录后接受绑定请求
5. 绑定成功后开始上传和沉淀记录

## 3. 主要页面说明

### 3.1 记录舱

- 上传图片、视频或文字内容
- 填写描述、感受等信息
- 保存到待处理区或归档区

### 3.2 待处理

- 查看尚未归档的内容
- 补充字段
- 编辑后归档

### 3.3 已归档

- 查看已经完成整理的个人记录
- 在符合规则时发起共享

### 3.4 情侣空间

- 查看当前已经对自己开放的伴侣内容
- 在允许范围内进行评论互动

### 3.5 账户

- 查看自己的用户信息
- 发起或处理绑定请求
- 导出自己的数据
- 发起解绑或确认解绑

## 4. 核心业务规则

### 4.1 默认私密

所有记录在创建后默认仅自己可见，不会自动对伴侣公开。

### 4.2 延时共享

即使用户主动申请共享，内容也不会立即对伴侣可见。系统会根据时间锁规则，在满足条件后才真正开放查看。

### 4.3 分手协议

解绑分为两类：

- 单方发起：进入冻结期，应用只读，双方可导出自己的数据
- 双方确认：直接清理关系相关数据

## 5. 常见流程

### 5.1 上传一条记录

1. 打开“记录舱”
2. 选择文件或输入文字
3. 填写必要字段
4. 保存

### 5.2 让一条记录进入共享流程

1. 打开“已归档”
2. 找到目标记录
3. 发起共享申请
4. 等待系统在满足规则后开放给伴侣

### 5.3 解除绑定

1. 打开“账户”
2. 进入解绑区域
3. 选择发起或确认解除
4. 根据系统状态完成冻结、导出或销毁流程

## 6. 常见问题

### 6.1 为什么对方看不到我写的内容？

因为记录默认私密。只有进入共享流程并满足规则后，对方才可见。

### 6.2 为什么我已经申请共享了，对方还是看不到？

申请共享不等于立即公开。系统会按照时间锁规则控制开放时机。

### 6.3 冻结期里还能做什么？

冻结期内系统以只读为主，但通常仍允许导出属于自己的内容。

### 6.4 没安装视频或图片处理依赖会怎样？

核心功能仍可使用，但某些预览能力可能降级。

## 7. 隐私提醒

- 不要把用户 ID、登录入口或运行环境暴露给无关人员
- 本项目当前主要面向本地或内网使用
- 在正式产品化前，不建议把它当作公网上的长期服务直接开放
