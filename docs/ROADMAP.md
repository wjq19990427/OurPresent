# OurPresent 上线路线图

**最后更新**：2026-05-15
**性质**：从当前 Alpha 走到商业化上市的阶段性计划。**关键作用是把"卡在用户决策 / 资源 / 法律手续"的事情显式列出来**，避免架构师默认等待、用户默认我在动。
**前置文档**：`docs/DIRECTION.md`（产品方向）·`docs/E2EE.md`（协议设计）

## 阶段概览

| 阶段 | 状态 | 卡点（谁来解锁） |
|------|------|----------------|
| Phase 0 · Alpha 本地 Demo | **当前** | — |
| Phase 1 · Alpha UX 收尾 | 随时可推 | — 我可独立驱动 |
| Phase 2 · 0→1 云端最小闭环 | **被资源卡住** | 用户：租云服务器 + 注册小程序 AppID |
| Phase 3 · 私测（体验版 + 真用户） | 待 Phase 2 | 用户：决定加入测试名单的情侣 + 评估 ICP 备案 |
| Phase 4 · 公测 / 上市 | 待 Phase 3 | 用户：**法人主体** + ICP 备案 + 隐私协议 / ToS 法务审 |

## Phase 0 · Alpha 本地 Demo（当前状态）

**做完的事**：

- Streamlit + SQLite 单进程，整库 dict 模型，71+ 测试
- 核心机制（延时共享 / 冻结期销毁 / 三视图）跑通
- 合成数据流水 + 角色卡 + 剧本（产物保留作未来回放基线）
- E2EE 协议设计已落定（`docs/E2EE.md`）
- 小程序事实边界已调研（`docs/research/wechat-miniprogram-crypto.md`）

**这阶段不会再增添：** AI 模块、PG 迁移、合成流水线扩展。所有这些都已冻结，等 Phase 2 重新决定形态。

## Phase 1 · Alpha UX 收尾（我可独立驱动，无外部依赖）

**目标**：在没有任何云端 / 小程序投入的情况下，把延时表达机制本身的 UX 打磨到"找两个真朋友坐下来用 10 分钟就能 get 到"的程度。

**为什么先做这个**：
- 延时表达是产品灵魂卖点，Phase 2 之后再回头改 UX 成本翻倍
- Streamlit 是当前最快的 UX 迭代载体；mini-program 工程一旦启动，每改一个交互都要走前后端
- 这阶段产出的 UX 决策直接转译成 mini-program 的设计稿，没有浪费

**候选工作**（我可以列详细任务卡）：
- 写入流程：感受 / 原因 / 解锁时间这几个字段的填写顺序与默认值
- 时间锁的可视化（90 天倒计时、解锁前 1 天提醒、即将解锁的 affordance）
- 解锁瞬间的 UX（已读回执？情绪标记？评论入口的可见度）
- 冻结期销毁的"提示 → 反悔 → 销毁"完整链路
- 三视图（我 / 我们 / 待解锁）的切换成本与信息密度
- 新用户引导（首次配对 → 第一条记录 → 第一次解锁的"魔法时刻"）

**完成判据**：你自己拉一个朋友坐下来不用解释就能上手。

## Phase 2 · 0→1 云端最小闭环（需要用户解锁基础设施）

**目标**：把 E2EE 协议从纸面跑到真机，证明 mini-program + 云端 + 密钥协商 + 密文存储 + 延迟派送的端到端链路可工作。

### 用户必须提供的事情

| 项目 | 估算成本 | 说明 |
|------|--------|------|
| 微信小程序 AppID | 免费 | `mp.weixin.qq.com` 注册，**个人主体**即可（无需公司），仅用于开发 |
| 云服务器（VPS） | ~30-100 元/月 | 选**境内云**（阿里云 / 腾讯云轻量服务器）以减少未来迁移成本；建议 2 核 2G 起步 |
| 域名 | ~50 元/年 | 用于服务端 HTTPS。境内云就必须搭配 ICP 备案，**境外云可暂时跳过备案**但小程序联调有限制 |
| ICP 备案 | 免费但耗时 | 个人备案可办（约 2-3 周）；公司备案要等法人主体。**Phase 2 内测可暂时绕过：用境外服务器 + 微信开发者工具的本地调试模式** |

**决策点**：境内云 + ICP 备案（前期慢但是为后续公测铺路） vs 境外云（开发阶段快，但 Phase 4 必须迁回境内）。

我倾向**先境外开发 / 后境内迁移**：Phase 2 / 3 的目标是验证机制，速度优先；Phase 4 商业化时再办备案 + 迁内地。代价是迁移有一次成本，但相比卡 2-3 周等备案是值得的。

### 我会驱动的事情

1. 创建 `OurPresent-miniprogram/` 工程（独立目录或独立仓库，待决策）
2. libsodium.js 在小程序上的 smoke test —— 把 task-23 报告里所有"未验证"项跑出真机结论
3. 配对协议端到端实现（按 `docs/E2EE.md` 第 2 节）
4. 服务端最小骨架：FastAPI / Node 任选，PG 存"密文 BLOB + 元数据"，延迟派送门禁
5. 一条用户故事跑通：A 写一条 90 天延时记录 → 服务端只存密文 → 90 天后服务端推给 B → B 解密看到

**完成判据**：你和我（或一个 trusted 伙伴）能在两台真机上把"延时共享 + 解锁"完整流程走通，且：
- 服务端数据库里抓出的内容字段全部是不可读密文
- 删除其中一方的本地存储 → 走伴侣协助恢复成功

### Phase 2 不做的事

- AI 模块（仍冻结）
- 复杂 UX（按最简能跑通即可，UX 在 Phase 3 打磨）
- 多设备同步（一人一台手机就够）
- 性能优化 / 监控 / 报警（够内测就行）

## Phase 3 · 私测（体验版 + 真用户）

**目标**：把 Phase 2 跑通的机制放给 5-10 对真情侣用 4-8 周，收集反馈。

### 用户必须解锁

- **加入测试名单的情侣**：微信小程序体验版限制最多 100 个测试者；你需要拉到 5-10 对愿意把私密记录交给一个内测产品的情侣（这一步可能比技术还难）
- **决定要不要办 ICP 备案**：Phase 3 时如果还在境外服务器，体验稳定性有限；办了境内 + ICP 体验上一个台阶。如果走个人备案这时可以启动
- **客服 / 反馈渠道**：内测期间一定会有数据丢失 / 配对失败 / 解锁异常等支持需求，你需要决定谁回复（你自己 / 我作为副本 / 还是自动化 FAQ）

### 我会驱动的事情

- UX 打磨（基于 Phase 1 决策 + Phase 2 真机反馈）
- 错误处理 / 文案 / 引导
- 监控基础（错误率、配对成功率、解锁成功率、数据丢失上报）
- 隐私协议 / 用户协议**初稿**（法务审在 Phase 4）

**完成判据**：5-10 对情侣里有 ≥ 60% 在 8 周后仍在用，且没有发生过"内容明文泄露"事件。

### Phase 3 不做的事

- 收费（仍免费）
- 公开宣传（仍口口相传）
- AI 模块

## Phase 4 · 公测 / 上市

**目标**：转商业化主体，向公众开放。

### 用户必须解锁

| 项目 | 说明 |
|------|------|
| **法人主体** | 公司或个体工商户。**这是无法绕开的**：小程序公开发布、商业化收款、ICP 备案到公司名下都需要 |
| ICP 备案（公司名下） | 如果 Phase 3 已经办了个人备案，到这一步要变更主体 |
| 微信小程序认证（公司主体） | 几百元/年，发布必需 |
| 公司银行账户 + 微信支付商户号 | 未来 AI 模块收费的基础 |
| 隐私协议 / 用户协议法务审 | 这是 Tier 1 隐私承诺真正变成法律护盾的关键步骤——文案里写的"我们也看不到"在法律上要站得住 |
| 客服 / 运营资源 | 公测后日活上 X 之后无法只靠你一个人回复 |

### 我会驱动的事情

- 性能 / 可扩展 / 监控加固
- 多设备同步实装（一个人多台设备共享 `K_couple`）
- 安全审计准备材料（如有第三方审）
- AI 模块**正式启动设计**（产品方向里推后的部分，到这阶段重新评估）

## 阶段间的"切换" checkpoint

每个 Phase 跨进下一阶段前，我会主动找你做一次 checkpoint：

- **0 → 1**：你给我"开始 Phase 1 UX 收尾"的信号即可，无外部依赖
- **1 → 2**：你确认基础设施到位（小程序 AppID 注册好、云服务器到手、决定境内/境外路线），我开新工程
- **2 → 3**：技术 demo 跑通后给你 review；同意后开始招测试者
- **3 → 4**：内测留存率达标 + 你完成法人主体 + 法务初审过

## 当前可立即推动的具体下一步

按重要性排序，**任你选**：

1. **启动 Phase 1**（无成本，立即开始）：我开始针对 Alpha Streamlit 上的延时表达 UX 列任务卡。这是无外部依赖、产出直接转译到 mini-program 的工作
2. **同步推进 Phase 2 基础设施准备**：你抽 1-2 小时去 `mp.weixin.qq.com` 注册个小程序 AppID（免费、用你自己微信号、个人主体），云服务器你方便时再租。这两件事办完就解锁 Phase 2
3. **决定 ICP 备案路线**：境内 vs 境外的选择越早定越好，影响 Phase 2 工程结构

我建议**1 + 2 并行**：我去推 Phase 1，你抽空办 AppID 和服务器。等你那边解锁了，Phase 1 也差不多收尾，可以丝滑切到 Phase 2。

## 我不知道但需要你告知的事情

1. **法人主体当前状态**：你已经有公司 / 个体户了吗？还是完全个人？这决定 Phase 4 的准备周期（公司注册要 4-6 周）
2. **预算量级**：每月愿意为基础设施花多少？影响服务器选型与 mini-program 认证时间
3. **上市目标时间感**：你心里的"上市"是 6 个月内 / 1 年 / 没定？这决定我现在是按"省成本"还是"省时间"模式来推进

这三件事不告诉我也不阻塞 Phase 1 / Phase 2 启动，但越早告诉我，越能让中长期计划准确。
