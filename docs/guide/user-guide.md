# OurPresent 用户使用文档

本文档面向最终使用者，介绍系统的安装、核心操作流程和常见问题。

## 1. 使用前准备

### 1.1 安装依赖

```bash
uv sync
```

### 1.2 配置 DeepSeek API（如需使用情感周报）

在项目根目录新建 `.env`：

```
DEEPSEEK_API_KEY=你的_key
```

未配置时，应用其他功能不受影响，仅周报无法生成。

### 1.3 启动应用

```bash
uv run streamlit run main.py
```

启动后访问 `http://localhost:8511`。

如果你希望让同一局域网中的另一台设备访问，也可以使用本机 IP 加端口访问。

## 2. 第一次使用

1. 双方分别注册自己的账号
2. 记录各自注册后显示的用户 ID
3. 一方在「⚙️ 设置」输入对方用户 ID 发起绑定
4. 另一方登录后接受绑定请求
5. 绑定成功后开始上传和沉淀记录

## 3. 主要页面说明

应用顶部三个 Tab：

### 3.1 🏠 我们（默认落地页）

- 双方已 `shared` 的记录按时间倒序展示
- 自己的卡片靠右，伴侣的卡片靠左（iMessage 风左右分边）
- 顶部「📊 周报」expander 展示最新情感周报（双方都开启服务后才生成）
- 详情区只读字段和文件，但评论区可双向互动
- 此 tab 不出现「编辑 / 申请共享 / 撤回」等自身记录操作——这些都在「📝 我的」

### 3.2 📝 我的

- 顶部「✍️ 写新记录」入口（上传文件或粘贴文字）
- 时间线展示当前用户全部 session，按 upload_time 倒序
- 每条卡片右上角带 4 态徽章：`[草稿] / [仅自己] / [倒计时·还有 N 天] / [已分享]`
- 详情区承载所有「我对自己记录的操作」：
  - 草稿：继续编辑字段、转归档
  - 仅自己 / 已分享：编辑字段、申请共享 / 撤回
  - 倒计时中：追加内容、改开放时间、立即解锁、撤回（修改时间 / 立即解锁需勾选二次确认）

### 3.3 ⚙️ 设置

- 当前用户资料
- 情感周报服务（开关 / 对方状态 / 频率 / 查看历史）
- 收到的绑定请求 / 伴侣绑定面板
- 解除绑定入口
- 冻结期数据导出入口

## 4. 核心业务规则

### 4.1 默认私密

所有记录在创建后默认仅自己可见，不会自动对伴侣公开。

### 4.2 延时共享时间锁

用户主动申请共享时，自选开放时间：
- 预设：立即 / 1 天后 / 3 天后 / 1 周后 / 1 个月后 / 90 天后
- 也可在日历中自定义任意日期
- 默认是 1 周

在等待解锁期间（`pending_unlock`），仍可对该记录：
- 追加内容（描述 / 感受 / 原因），原文保留 + "追加于 时间" 标记
- 修改开放时间（需勾选二次确认）
- 立即解锁（需勾选二次确认，明确告知会改变对方看见时刻）
- 撤回共享申请，恢复为 `private`

### 4.3 情感周报

- 双方都需在「⚙️ 设置」里开启「情感周报服务」开关；任一方未开启则不生成
- 双方都开启后，频率可在「设置」里调整（7 / 14 / 30 天），默认 7 天
- 周报基于 **已共享内容** 生成，不读私密记录
- 含四模块：关系足迹（计数）/ 情绪气象（情绪标签 + 天气式陈述）/ 同频共鸣（同日共享的短语）/ 未尽悬念（未解锁倒计时）
- 数据较少时（窗口内 < 3 条共享）生成「sparse」报告，仅展示足迹与一句温和说明
- 历史报告：在「⚙️ 设置 → 情感周报服务 → 查看周报历史」中查看
- 冻结期不生成新报告，历史仍可读

### 4.4 分手协议

解绑分为两类：

- 单方发起：进入冻结期，应用只读，双方可导出自己的数据
- 双方确认：直接清理关系相关数据（含历史周报）

## 5. 常见流程

### 5.1 上传一条记录

1. 进入「📝 我的」
2. 点击顶部「✍️ 写新记录」
3. 选择文件或输入文字
4. 填写必要字段（描述必填，文件型记录的描述也必填）
5. 选择「完成」（归档）或「存为草稿」

### 5.2 让一条记录进入共享流程

1. 进入「📝 我的」
2. 找到目标记录，点击展开详情
3. 「申请共享」选择开放时间档位或自定义日期
4. 提交后记录变为「倒计时」状态，可见伴侣需要等到开放时间

### 5.3 启用情感周报

1. 进入「⚙️ 设置」
2. 找到「📊 情感周报服务」section
3. 勾选「开启我的情感周报服务」
4. 让伴侣也开启
5. 双方都开启后，可一起约定频率
6. 系统会按节律自动生成；积累 ≥ 3 条共享记录后第一份周报才会有完整内容

### 5.4 解除绑定

1. 进入「⚙️ 设置」
2. 找到解绑入口
3. 选择「发起解绑」或「双方确认立即解绑」
4. 根据系统状态完成冻结、导出或销毁流程

## 6. 常见问题

### 6.1 为什么对方看不到我写的内容？

因为记录默认私密。只有进入共享流程并到达 `unlock_at` 时间后，对方才可见。

### 6.2 为什么我已经申请共享了，对方还是看不到？

申请共享时你选择了开放时间。如果选的不是「立即」，需要等到 `unlock_at` 才会自动开放。可在「📝 我的」找到该记录，看「倒计时·还有 N 天」徽章；也可主动「立即解锁」。

### 6.3 情感周报为什么没生成？

最常见原因：
1. 任一方没开启「情感周报服务」开关
2. 双方都开启但还没攒够 3 条共享记录，已生成的是 `sparse` 报告，只显示足迹
3. 上次生成失败（如 API key 错），下次自动重试一次

### 6.4 周报会读我的私密记录吗？

不会。生成只走 `visibility == "shared"` 的记录。`private` / `pending_unlock` 永远不进入周报 pipeline。

### 6.5 冻结期里还能做什么？

冻结期内系统以只读为主，但允许：
- 导出属于自己的内容
- 查看已经生成的历史周报
- 不会生成新的周报

### 6.6 没安装视频或图片处理依赖会怎样？

核心功能仍可使用，但某些预览能力可能降级。

## 7. 隐私提醒

- 不要把用户 ID、登录入口或运行环境暴露给无关人员
- 本项目当前主要面向本地或内网使用
- `.env` 中的 `DEEPSEEK_API_KEY` 不入库（已加入 `.gitignore`）；不要把它复制到聊天工具或公共仓库
- 在正式产品化前，不建议把它当作公网上的长期服务直接开放
